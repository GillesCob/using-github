  1. Je créé une branche
  2. Je fais mes modifs puis je les commit
  3. Je créé un pull request avec le détail de toutes mes modifs
  4. Ce code est vérifié par le N+1, commenté et à moi d'apporter les modifs
  5. Une fois validées, mes modifs sont ajoutées au main

Modifications du code

On va prendre l'habitude de travailler en local avec VS Code installé sur notre machine.

Nous n'allons pour le moment pas utiliser de branches.

Etape 1 : Ouvrir son code en local
  • Code => Local => Open with GitHub desktop
  • Ouvrir VS Code depuis GitHub desktop
  • Réaliser les modifications
Etape 2 : Tracker les modifications
  • structurer ses modifications par div
  • [git add "nom fichier qui contient les modifs"]
  • [git commit -m "description des modifs"]
  • recommencer au fil des modifs et des fichiers
Etape 3 : Savoir où on en est
  • Pour savoir où on en est : [git status]
  • Possibilité de vérifier les logs : [git log]
  • Si on veut plus de détails : [git log --stat]
Etape 4 : Diffuser
  • [git push origin main]

Utilisation avancée

Voici quelques éléments en cas de fonctionnement en équipe :

Etape 1 : Importer les MAJ du main
  • C'est le cas si d'autres dev ont eu leurs MAJ incoirporés au main : [git checkout main]
  • On récupère ces MAJ : [git pull]
  • On vérifie les MAJ présentes dans notre code : [git checkout "nom_branch"]
  • On intègre les MAJ du main dans notre code : [git merge main]
  • On pousse nos changements : [git push]
